# Архитектура (черновик)

## Основные сущности
- User: Telegram user (Илья/Настя).
- Entry: запись минут медитации (`minutes`, `created_at`, `user_id`).

## Поток данных
1. Пользователь отправляет `+N`.
2. Handler валидирует формат и доступ пользователя.
3. Service сохраняет запись в SQLite.
4. Service рассчитывает агрегаты за день/неделю/месяц.
5. Bot отвечает суммой и, по расписанию, шлет итоговый отчет.

## Модули
- `config.py`: загрузка переменных окружения.
- `db/`: доступ к SQLite.
- `services/tracker.py`: бизнес-логика учета и агрегации.
- `handlers/`: обработчики команд/сообщений.
- `main.py`: точка входа и запуск polling.
